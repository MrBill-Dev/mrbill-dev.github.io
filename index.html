📚 AI 實戰學程

從日常工作情境出發，逐步建立 AI 協作能力，涵蓋 內容整理、寫作設計、開發自動化 與影像生成流程。

導讀前言

現在的 AI 工具很多，新手最容易卡住的，不是不會用，而是不知道該先學哪一個。

其實不用一開始就研究所有模型，也不用急著把全部功能學完。先選一套工具、完成一個實際成果，比同時學很多東西更重要。

這個學程會從日常最常見的工作情境開始，帶你理解不同 AI 的用途與差異，包含內容整理、寫作、設計、開發、自動化到影像生成等流程。你會慢慢知道：

  • 哪種 AI 適合哪種工作
  • 怎麼把問題問得更清楚
  • 哪些地方需要人工判斷
  • 怎麼把 AI 接進真實工作流程

如果你是第一次接觸 AI，建議先從「AI 差異比較」與「Prompt 基礎」開始。先學會把需求說清楚，再往後學 RAG、Agent、多模態等進階應用。

不用急著一次看完整份課綱。先完成第一個可交付成果，你會更容易理解後面的內容到底在解決什麼問題。

🧭 新手先看：你每天在用的 AI，到底差在哪？

ChatGPT

通用問答、寫作、整理資訊很快，適合日常工作與學習。

Claude

長文分析與結構化整理穩定，適合讀文件、寫教案。

Gemini

整合 Google 生態與搜尋優勢，適合文件、雲端流程與研究摘要。

通義 / 豆包 / Kimi / 文心

中文語境與在地平台整合常較友善，適合特定地區工作流需求。

Midjourney / DALL·E / Flux / SD

影像生成精準度與風格控制表現強，適合主視覺發想；商用仍需人工審稿與授權確認。

Cursor / Copilot 類

IDE 協作最有感：可結合專案脈絡做改碼、重構、除錯與測試建議。

Whisper / ElevenLabs / Runway

偏語音與影片工作流：字幕、配音、短片合成，適合多媒體交付。

開源本機模型（Llama / Qwen 等）

重隱私與可控性，適合內部資料；但建置與硬體門檻較高。

💳 免費版 vs 付費版（新手必懂）

  • 免費版：速度與配額受限、尖峰時段較不穩。
  • 付費版：通常更快、更穩，長文與高階模型可用性較高。
  • 精度差異常來自「模型等級 + 可用上下文長度 + 工具功能」。

🖥️ 本機架設 vs ☁️ 雲端 AI

  • 本機：資料可控、離線可用，但需 GPU/記憶體與部署維護能力。
  • 雲端：上手快、模型更新快，適合大多數新手與團隊。
  • 準確度不只看本機或雲端，還要看模型品質、資料新鮮度與提示詞設計。

新手重點：不是「哪個 AI 最強」，而是「你會不會問」。

你現在常用的「一句話 ask」可以先用，但接下來要學的是：如何把同一個問題，逐步重問成可交付結果。

❌ 錯誤示範：模糊 ask

幫我寫一篇 SEO 文章
  • 沒有受眾：寫給誰？
  • 沒有目標：要流量還是詢單？
  • 沒有格式：長度與輸出形式不明。

✅ 正確示範：結構化 ask

你是內容編輯。請寫一篇給「手工皂新手」的文章。
目標：提升頁面 CTA 點擊率。
格式：1200 字、H2/H3 架構、文末 FAQ 3 題。
語氣：白話、可執行。
最後附：一段社群貼文版摘要。

同樣是問「寫文章」，加上角色、受眾、目標、格式，結果品質會直接提升。

🔁 一般 ask 大法（重問流程）

第 1 問

先拿草稿（不要期待完美）。

第 2 問

要求改成指定格式（條列、表格、步驟）。

第 3 問

補上你的情境資料（客群、商品、時間、限制）。

第 4 問

要求「錯誤檢查＋可執行清單」再交付。

🧩 新手四階段：學前錯誤 → 學後能力 → 下一步

階段 1：只會一句 ask

學前常錯：「幫我寫一篇文章」沒有對象、目標、格式。

學後會：寫出角色＋受眾＋目標＋輸出格式的完整提問。

下一步：到 Prompt 工具 練 4 段式模板。

階段 2：只看答案，不驗證

學前常錯：AI 回什麼就直接貼上去使用。

學後會：用對照、查證、重問，把答案修成可交付版本。

下一步：到 互動練習 做查證題與改寫題。

階段 3：會問，但不會落地

學前常錯：有文字結果，但沒有對應網站/設計/行銷情境。

學後會：把 AI 產出落到頁面、素材、簡報或專案流程中。

下一步：去 前端手冊、視覺設計學院 做實作。

階段 4：有產出，沒有成效回看

學前常錯：作品做完就結束，沒有追蹤是否有效。

學後會：用 SEO/GEO/GA4 指標回看，知道該改哪一步。

下一步：到 SEO / GEO 實戰 跑優化循環。

M3 · RAG 與企業知識應用 進度：中階

文件切片、向量搜尋、引用回答。客服 FAQ、內部 SOP、法規查詢——避免把機密資料丟進公開免費版。

下一步去哪：練習箱 RAG 題 + NotebookLM 小實驗

需要工具：NotebookLM 或企業知識庫工具、可上傳 PDF

前置基礎：建議先完成 M1~M2；0 基礎可做概念版

M4 · AI Agent 與自動化 進度：中階

任務拆解、工具呼叫（搜尋、API、檔案）。理解 Cursor、Copilot、Zapier+AI 的邊界與風險控管。

實務：定義「人類必須審核」的關卡清單

下一步去哪：小遊戲情境決策（先練「哪些不能全自動」）

需要工具：Cursor / Copilot / Zapier（三選一先開始）

前置基礎：建議先有 M2；0 基礎先做流程圖與權限邊界

M5 · 開發協作（Vibe Coding / Code Review） 進度：中階

AI 生成 HTML/CSS/JS、單元測試、重構建議。人類負責架構、安全、RWD、無障礙與 GA4 事件。

實務：前端手冊 + IDE 外掛

下一步去哪：前端手冊（每課至少跑一次沙盒）

需要工具：VS Code / Cursor、瀏覽器 DevTools、Git 基本指令

前置基礎：HTML/CSS/JS 基礎；0 基礎先補手冊入門課

M6 · 多模態：影像、影片、語音 進度：中階

Midjourney / Flux 提示詞、Runway / Kling 短片、Whisper 字幕、TTS 配音。先懂攝影曝光再生成，品質才可控。

實務：攝影學堂

下一步去哪：攝影學堂 + 影像創作 Prompt

需要工具：MJ / DALL·E / Runway 擇一，手機或相機做對照拍攝

前置基礎：建議先懂曝光三角；0 基礎可先做手機路線

M7 · 行銷、SEO 與 GEO 進度：可交付

AI 輔助內容產出 + 人工 E-E-A-T；結構化段落利於 Google 與 AI 摘要引用；GA4 驗證轉換。

實務：SEO / GEO 實戰

下一步去哪：SEO / GEO 實戰頁（跑一輪內容優化）

需要工具：Search Console、GA4、內容編輯器

前置基礎：M2 結構化提問必備；0 基礎可先做文案流程，不做技術 SEO

M8 · 治理、倫理與合規 進度：可交付

幻覺查證、個資法、版權、偏見、深度偽造標示。企業採用 ChatGPT Team / Azure OpenAI 的決策要點。

實務：概念練習「倫理與安全」· 小遊戲「幻覺辨識」

下一步去哪：倫理安全練習 + 幻覺辨識遊戲

需要工具：查證來源、企業資料分級清單、法務或管理規範文件

前置基礎：所有階段都應並行學；0 基礎也必學

📰 AI 大記事

截至 2026/05/28：近期更新改版重點與實務影響。

OpenAI / Google / Anthropic：多模態能力成主流
  • 更新重點：ChatGPT（OpenAI）、Gemini（Google）、Claude（Anthropic）都持續強化文字+圖片+語音整合。
  • 影響：同一任務可以在一個對話中完成草稿、改寫、素材建議與整理，不必頻繁換工具。
Cursor / GitHub Copilot / OpenAI：Agent 與 IDE 協作升級
  • 更新重點：以 Cursor、GitHub Copilot、OpenAI Agent 類流程為代表，多步驟執行與專案上下文能力更成熟。
  • 影響：能更快完成「需求→實作→修正」，但需要明確的人類審稿與版本節點。
Midjourney / DALL·E / Adobe Firefly / Flux：產圖精準度提升
  • 更新重點：Midjourney、DALL·E、Firefly、Flux 在風格一致、細節控制、排版可用性上持續進步。
  • 影響：「只要做圖」的使用者可先出主視覺，再自行補字；但仍要做授權與品牌一致性檢查。
免費版 vs 付費版：ChatGPT / Claude / Gemini 差在哪
  • 更新重點：付費方案通常優先開放較強模型、更高使用額度與更長上下文。
  • 影響：若你常遇到「重跑很多次還失敗」，通常要先優化提示詞，再評估是否升級付費版。
各家額度與速率限制調整
  • 更新重點：OpenAI、Anthropic、Google 近年都更頻繁調整免費/付費額度、尖峰速率限制與工具可用次數。
  • 影響：同一提示詞在不同時段可能失敗率不同；建議保留備援模型與離峰工作習慣。
Meta Llama / Mistral / Qwen：本機模型 vs 雲端模型
  • 更新重點：Llama、Mistral、Qwen 等開源系模型讓本機部署更可行，雲端則持續擴大工具整合速度。
  • 影響：有隱私需求或固定流程可考慮本機；追求最新能力與省維運通常先用雲端。
Microsoft / Google / OpenAI：企業導入轉向治理與可追溯
  • 更新重點：企業方案普遍強化資料隔離、權限分層、記錄稽核與引用追蹤。
  • 影響：工作場景不只要快，還要能被查證；這會直接影響你選哪個模型與流程設計。

AI 協作怎麼做

先看這三步，你就知道怎麼跟站內工具搭配。

1. 先定任務

先選只要發文、只要出圖，或只要修 Prompt。

2. 用站內工具先出初稿

文案到 `內容表達學`，出圖到 `視覺設計學院`。

3. 回 AI 區做精修

在 AI 頁學模型選擇與 Prompt 修正，降低重跑失敗率。
